# Support

This document explains how to get help with Project Production Planning, a
Microsoft Dynamics 365 Business Central extension maintained by UsualApps Inc.

## Bugs and Feature Requests

Use this repository's [GitHub issues](https://github.com/devhs89/UsualApps.ProjectProductionPlanning/issues)
to report reproducible application defects or propose enhancements. Before
opening an issue, search existing issues to avoid duplicates.

Include the following information when reporting a bug:

- Project Production Planning version.
- Business Central application version and environment type.
- A clear description of the expected and actual behavior.
- Steps to reproduce the problem.
- Relevant configuration, error messages, and logs.
- Screenshots when they help explain the issue.

Remove credentials, license files, customer information, personal data, and
other sensitive content before submitting anything publicly.

## Usage and Configuration Help

For help using or configuring the extension, contact UsualApps through the
[UsualApps contact form](https://usualapps.com/contact/). Describe the problem,
the affected version, and the Business Central environment, but do not include
secrets or sensitive customer data in the initial message.

Support availability, response times, and services depend on any agreement you
have with UsualApps Inc. Submission of an issue or request does not guarantee a
particular response time or resolution.

## Security Vulnerabilities

Do not report suspected security vulnerabilities in a public issue. Follow the
private reporting instructions in [SECURITY.md](SECURITY.md).

## AL-Go and Third-Party Tooling

This repository uses [Microsoft AL-Go for GitHub](https://github.com/microsoft/AL-Go)
for build and release automation. Report problems in this repository's AL-Go
configuration here. Report defects originating in the upstream AL-Go tooling to
the Microsoft AL-Go project.

For other third-party products or dependencies, use the support channel
provided by the relevant maintainer.
